# プロジェクト状況

- 言語: [EN](PROJECT_STATUS.md) | [KO](PROJECT_STATUS.ko.md) | **JA**

## プロジェクトの位置づけ
MuseTalk-RTは、MuseTalk派生ランタイム候補を製品化に近づけるための検証・証拠化レイヤーです。

## 何が完了し、何が未完了か

### 完了
1. Warm-pathポリシーを運用化
2. ライフサイクル検証チェーンを整備
3. 4主要ステージをflagged経路で provisional real-wired 化
4. 30-run 信頼性で fallback 0 / stage error 0
5. bursty/jittery ストレス下でもライフサイクル安定性を維持
6. 安定/劣化を識別できる品質テレメトリを整備
7. Human QAパック、スコアカード、判定レイヤーを整備
8. ローカルGUIと多言語ドキュメントを整備

### 未完了
1. 実レビュアー評価の継続収集
2. カナリア判断用エビデンスの完成
3. モバイル/ネットワーク劣化想定での知覚品質エビデンス
4. より広い製品準備性ゲートの明確化

## カナリア準備完了とみなす条件
- stable/bursty/jittery の代表サンプルに対する人手スコアが揃っている
- 判定レイヤーで `GO_WARN_ONLY_CANARY` を満たす
- 直近の信頼性指標に退行がない
- ロールバック条件と責任者が明文化されている

## より広い製品準備完了とみなす条件
- 複数回・複数評価者で再現する品質証拠
- 評価者間整合（ばらつきの管理）
- 実運用観点の監視/ロールバック運用性の確認
- flagged 非本番範囲を超えることへの合意形成
